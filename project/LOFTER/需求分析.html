需求分析

lofter 是国内主要的图片为主的轻博客应用. 包含web 移动端

lofter 的前生是网易博客, 所以要保留博客的性质. 同时增加差异化的竞争力, 突出图片社交的性质.

改版动机

• 向竞品学习, 提高产品竞争力

• 通过用户反馈, 和个人体验, 改善交互设计

• 突出业务需求

• 个人的技能实训

业务目的

• 网易博客向lofter改版过渡

• 提高市场竞争力

• 提高用户体验, 增强差异竞争力

业务限制

• 保持网易统一的风格

• 推广网易其他产品

由于是改版设计, 按照原来的lofter的样子, 分析下用户的特点

目标用户

• 网易博客老用户

• 摄影爱好者

• 乐于总结分享的行业专家

• 自媒体

• 乐于发现, 猎奇的年轻人

用户特点

• 专业能力强, 通过分享获得关注

• 作者和读者, 较风化

• 通过兴趣联结, 陌生人社交

• 个人博客空间

用户场景

• 博客老用户, 以及习惯写博客的人, 在app以及web端创作

用户行为

• 创作

• 阅读

• 使用服务

• 互动

衡量指标

• 总创建量

• 总阅读量

• app下载, 留存

• 等等

• 领域专家在以博客的形式记录自己的经验, 学习笔记, 生活感悟等等

• 手机摄影爱好者, 拍照/修图 后发布到lofter分享, 移动端为主

• 专业摄影爱好者, 拍照/修图 后发布到lofter分享, web端为主

• 自媒体, 定期发布信息, 吸引关注

• 读者, 利用琐碎时间, 在移动端 或者web 端阅读 博文 或者 图片

用户行为(移动端)

• 在移动端发轻博客

• 在移动端发送图片, 适度的再编辑

• 管理自己的博文

• 推广自己吸引关注

• 碎片的阅读, 对感兴趣的内容快速整理

用户体验目标(移动端)

• 快速创作

• 快速整理

• 第一时间获得关注反馈

• 发图前适当编辑

• 通过分享, 直接发布

• 粉丝推送服务

分析发现, 作为一个平台 有很多不同类型的使用者, 最终确定的方案要权衡这些不同类型用户的需求

分解关键因素

动机

• 社交需要, 发现同好

• 交流分享

• 个人记录

• 自媒体

• 发现 猎奇 寻找信息

担忧

• 版权 隐私问题

• 编辑保存

• 流量啊流量

障碍

• 网络状态不好

• 功能认知

初步解决方案

• 增强社交性, 兴趣发现, 扩散关注

• 强调分享

• 版权隐私管理

• 推广

• 监测网络环境, 选择性加载不同图片质量

• 自动保存为草稿
